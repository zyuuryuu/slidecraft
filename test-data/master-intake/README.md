# master-intake 実機テスト用データ一式

F1（任意スライドマスター対応）の改善を実機で体感するためのテンプレ＋原稿＋チェックリスト。
すべて合成物（IP 非含有）。生成器は [`scripts/make-test-templates.ts`](../../scripts/make-test-templates.ts)。

## 使い方

1. アプリを起動（`npm run tauri dev`、または `npm run dev` でブラウザ）。
2. **テンプレートを取り込む**：`messy-corporate.pptx` を「テンプレート読み込み」で開く。
3. **原稿を投入する**：`sample-deck.md` の中身を Markdown 入力（貼り付け or ファイル）で流し込む。
4. プレビューを見て、下のチェックリストを確認する。

各スライドは `<!-- slide: レイアウト名 -->` でレイアウトを固定してあるので、狙ったレイアウトの
束縛を直接確認できる。

## テンプレ①：`messy-corporate.pptx`（見た目は普通・構造が雑）

「見た目は普通の会社デッキだが、見出しが body 型・running header 付き・placeholder 名が汎用」
という実運用で頻出する乱雑さを仕込んである。**F1 以前ならタイトルが迷子になり、本文がヘッダーや
フッターに漏れていた**ケース。

### チェックリスト

- [ ] **表紙**：`2026年度 事業戦略` が大見出し、`経営企画部` がサブ。（クリーンなレイアウト）
- [ ] **本文（ヘッダー付き）** ← 目玉
  - [ ] 見出し `市場環境の変化` が上部の見出し枠に入る
        （その枠は **body 型・名前も「テキスト プレースホルダー」** なのに、幾何から title と推定＝**②b 復元**）
  - [ ] 上端の細いヘッダー帯（`hdr` 型）に **本文が流れ込まない**（＝chrome 硬除外）
  - [ ] 3つの箇条書きが本文枠に入る
  - [ ] 下部のフッター／日付／ページ番号に **本文が漏れない**
- [ ] **本文（幅広ヘッダー）** ← #96 の再現レイアウト
  - [ ] 見出し `市場環境の変化` が見出し枠に入る（**タイトルが迷子にならない**）
  - [ ] 本文が **見出し枠へ1つズレず**、本文枠に入る
  - [ ] スライド幅いっぱいの running header 帯（`hdr` 型）に何も入らない
  > 上の「本文（ヘッダー付き）」と同型で、ヘッダー帯が**幅広**なだけ。#96 修正前は、この幅
  > （スライド幅の 55% 以上）だと帯が title 枠と誤認され、見出しの復元が止まり、タイトルが
  > 未束縛・本文が見出し枠へズレていた。
- [ ] **3カラム**：見出しが上部・本文が段組領域に入る（見出し枠は body 型だが title に）
- [ ] **図＋説明**：見出しが上部・左の大きな枠に図、右が注記
  > 注：この枠の「figure 判定」は **binding には効いていない**（散文も入る）。scorer の figure は
  > 現状 precision が低く、健全テンプレの本文枠まで figure と誤判定するため配線していない（#98）。

> F1 以前の挙動（参考）：見出しは title 枠が無いため body 扱いのまま → deck の
> タイトルが**行き場を失う**か、running header/footer に**本文が誤注入**されていた。

## テンプレ③（任意）：`lrk-slides-velis_CC0.pptx`（実在の A4 テンプレ・CC0）

リポジトリの [`tests/fixtures/templates/lrk-slides-velis_CC0.pptx`](../../tests/fixtures/templates/lrk-slides-velis_CC0.pptx)。
唯一の実世界 outlier（**A4・複数マスター・型なし placeholder 多数**）。非16:9 でも幾何判定が
崩れない（F0b）ことの確認用。

- [ ] レイアウトが破綻せず読み込める・タイトル/本文の枠が妥当に選ばれる

## 見るべき勘所（まとめ）

| 改善 | どこで見えるか |
|---|---|
| **header 誤注入の根治**（②b ゲート）| ヘッダー帯・フッターに本文が入らない |
| **幅広 header の title 誤認 根治**（#96）| 「本文（幅広ヘッダー）」で見出しが復元・本文がズレない |
| **body 見出し → title 復元**（②b）| 「テキスト プレースホルダー」名の枠に見出しが入る |
| **chrome 硬除外**（②a/②b）| 日付・番号・フッターが本文で潰れない |
| **非16:9 対応**（F0b）| velis(A4) が破綻しない |

気づいた点（うまくいかない・意図と違う配置など）があれば教えてください。特に「本文が意図しない
枠に入る」「見出しが拾われない」は最優先で調べます。
