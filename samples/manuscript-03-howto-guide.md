# 新メンバーの開発環境セットアップガイド

このドキュメントは、新しくチームに参加したエンジニアが、最初の一日でローカル開発環境を立ち上げ、最初のプルリクエストを出せるようになるまでの道のりをまとめたものです。手順そのものよりも「なぜそうするのか」を理解してもらうことを大切にしているので、つまずいたときは遠慮なくチームの誰かに声をかけてください。私たちは質問を歓迎する文化を持っています。

## はじめに

私たちのプロダクトはデスクトップアプリケーションであり、ビルドにはいくつかの外部ツールチェーンが必要です。最初は依存関係が多く感じるかもしれませんが、一度そろえてしまえば日々の開発は驚くほど快適になります。焦らず、一歩ずつ進めていきましょう。所要時間はおよそ半日を見込んでおくと安心です。

## 環境を整える

まずは土台となるツール群をインストールします。順番が前後すると後の手順でエラーが出ることがあるため、次の流れに沿って進めてください。

1. まず社内アカウントを発行してもらい、GitHub Organization への招待を受け取ります。招待メールが届いていない場合はオンボーディング担当に連絡してください。
2. 次に Node.js と Rust のツールチェーンをインストールします。バージョンはリポジトリ直下の設定ファイルに固定されているので、それに合わせてください。
3. 続いてリポジトリをクローンし、依存パッケージをインストールします。ここで時間がかかるのは正常です。
4. 最後に環境変数のテンプレートをコピーし、必要な値を自分用に書き換えます。

ここまで終えると、ローカルでアプリを起動する準備が整います。

## 動作を確認する

セットアップが正しく完了したかは、実際に動かして確かめるのが一番確実です。開発サーバーを起動し、ブラウザでプレビュー画面が表示されることを確認してください。あわせてテストスイートを一度走らせ、すべて緑になることを見届けます。もしテストが落ちる場合、それは多くの場合あなたのミスではなく、環境のどこかが噛み合っていないサインです。エラーメッセージをそのままチームに共有してくれれば、原因をすぐに切り分けられます。

## 最初の一歩

環境が動いたら、ウォームアップとして小さな修正に挑戦してみましょう。タイポの修正やコメントの追記など、ごく軽いもので構いません。ブランチを切り、変更をコミットし、プルリクエストを作成する。この一連の流れを体験することが、チームの開発リズムをつかむ何よりの近道です。最初のレビューを楽しみにしています。
