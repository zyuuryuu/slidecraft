# Git ブランチ戦略

## ブランチ命名

`claude/<topic>-<session-id>`

session-id は自動付与。topic は英語で簡潔に。

## ブランチ判断マトリクス

**別ブランチで作業**:
- 独立した機能追加（新テーマ、新アイコン、新テストスイート）
- Sub Agent に委任するタスク
- P6 の E2E テスト / インストーラ作業

**main 直接作業**:
- engine/ 内の複数ファイルにまたがる型変更
- schema.ts の変更を伴うリファクタ
- 軽微な修正（typo、コメント修正）

## マージプロトコル

1. PR 推奨（履歴保持のため）
2. 軽微なフォローアップは直接マージ可
3. マージ前に diff 確認: `git diff HEAD origin/<branch> --stat`
4. コンフリクト発生時はユーザに確認
5. マージ後はリモート・ローカル両方のブランチを削除
