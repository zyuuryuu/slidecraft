# ADR-0034: 便利スライドの生成 — live/static ハイブリッド（目次ほか）

- Status: Accepted
- Date: 2026-07-20

## Context

目次・アジェンダ・章扉・まとめといった「既存コンテンツから導ける便利スライド」を、今は
`<!-- toc -->` / `<!-- section -->` の**特殊コメント記法**＋レンダー時の `materializeDerivedSlides`
（[ADR-0032](0032-authoring-notes-and-sections.md)）で live 導出している。これには2つの摩擦がある：

1. **利用者が特殊記法を手書きする必要**がある（`<!-- toc -->` を知らないと目次が作れない）。
2. **live 導出は消費点ごとに materialize を呼ぶ規約**で、呼び忘れると表示がずれる隠れた脆さがある
   （実例：左サムネイル一覧が materialize を通さず目次が空表示になった #275）。

一方 ADR-0032 D2 は「便利スライドの**自動生成**」を意図的に却下していた。理由は実装量でなく構造：
ADR-0002（deck が唯一の源）の下では生成スライドも GUI で見え**編集できてしまう**が、その編集を
コメント1行へ**畳み戻せない**——保持すればデタッチ規則という新問題クラス、破棄すれば
no-silent-drop（ADR-0030）違反。だから「著者が書く章扉＋タグ」に倒し、目次だけ導出専用にした。

利用者フィードバック（2026-07-20・実機）：「目次生成のような便利スライドを既存コンテンツから
生成する仕組みを入れれば、こういう特殊記法・処理が要らなくなるのでは」。

## Decision

**GUI 駆動の「便利スライド生成」を導入する。** 各タイプは生成時に**2モードから選ぶ**ことで、
ADR-0032 が突き当たったジレンマを「どちらを取るかを利用者の明示選択にする」ことで解消する：

- **live（自動更新）** — drift-proof な導出スライド。ADR-0032 の `derived` 属性＋`materializeDerivedSlides`
  機構をそのまま使う。**直接編集不可・章に自動追随**。Markdown 往復は従来のマーカーが担う（ただし
  マーカーは**利用者が手書きする記法ではなく、live スライドの直列化形**に降格＝GUI 生成が正面口）。
  **目次の既定は live**（目次が古くなるのは実務で害が大きい）。
- **static（固定）** — 現在のコンテンツから1回生成した**普通の編集可能スライド**。特殊処理ゼロ・
  DeckIR の通常スライドとして往復。「コンテンツから作り直す」明示アクションを備える（古くなったら
  利用者が更新＝黙って drift しない）。**畳み戻しジレンマが存在しない**（普通のスライドとして畳まる）。

初期タイプは**目次**。アジェンダ再掲・章扉・まとめ等は後続タイプとして同じ枠に足す。

### なぜこれが ADR-0032 のジレンマを解くか

ADR-0032 D2 の却下理由＝「編集可能な生成スライドを畳み戻せない」。本 ADR は
**static＝畳み戻す必要のない普通のスライド**（生成器は一度きり・以後は通常編集／明示再生成）とし、
**live＝そもそも編集不可の導出**（畳み戻し対象が無い）とする。両モードとも畳み戻しジレンマに
入らない。ADR-0032 が「生成 or 導出」の二択を迫られ導出だけを選んだのに対し、本 ADR は
**両方を用途で選べる**ようにした。

### やらないこと / 不変条件

- ADR-0002 は維持：static は通常の deck スライド、live は複製状態を持たない導出（R8）のまま。
- 手書きの `<!-- toc -->` / `<!-- section -->` は**後方互換で読み続ける**（既存デッキ・md 入力を壊さない）。
  変わるのは「利用者が手書きしなくても GUI 生成で作れる」点＝記法は必須でなくなる。
- 新記法を含まない Markdown の parse→export は byte-identical（ADR-0032 と同じ不変条件を継承）。
- live スライドの表示一致は #275（表示用 materialize deck の一本化）で担保。

## Consequences

- (+) 「既存コンテンツから便利スライドを生成」という素直な UX。目次以外にも拡張しやすい枠。
- (+) 特殊コメント**記法**を利用者の必須知識から外せる（GUI が正面口・マーカーは往復の内部形）。
- (+) drift-proof（live）と編集自由（static）を**用途で選べる**。目次は live 既定で乖離しない。
- (+) ADR-0032 D2 の「生成却下」の構造的理由を、モード選択で解消。
- (−) 生成 UI（メニュー＋live/static 選択）と、static の「作り直し」アクション＝新規実装。
- (−) live/static の2状態が増える＝各タイプで往復・表示・編集の扱いをテストで固定する必要。
- **ADR-0032 D2 を部分 supersede**（「便利スライドの生成は行わない」の方針のみ。live 導出機構・
  `<!-- note -->` ノート・章タグ採番は存続）。**ADR-0002 は不変**（生成物の扱いを上記で両立）。
